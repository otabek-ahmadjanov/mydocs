# finalize() method

Метод `finalize()` называется финализатором.

Финализаторы вызываются, когда JVM определяет, что данный экземпляр должен быть удален сборщиком мусора. Основная цель финализатора — освобождение ресурсов, используемых объектами, до их удаления из памяти.

Ниже приведен пример финализатора

```java
public class Finalizable {
    private BufferedReader reader;

    public Finalizable() {
        InputStream input = this.getClass()
                .getClassLoader()
                .getResourceAsStream("file.txt");
        this.reader = new BufferedReader(new InputStreamReader(input));
    }

    public String readFirstLine() throws IOException {
        String firstLine = reader.readLine();
        return firstLine;
    }

    @Override
    public void finalize() {
        try {
            reader.close();
            System.out.println("Closed BufferedReader in the finalizer");
        } catch (IOException e) {
            // ...
        }
    }
}
```

Время, когда сборщик мусора вызывает финализаторы, зависит от реализации JVM и условий системы, которые находятся вне нашего контроля.

**`Недостатки финализаторов:`**

- Мы не можем знать, когда запускается финализатор, поскольку сборка мусора может произойти в любой момент. Само по себе это не проблема, поскольку финализатор всё равно будет выполнен рано или поздно. Однако системные ресурсы не безграничны. Таким образом, ресурсы могут исчерпаться до того, как произойдёт очистка, что может привести к сбою системы.
- Отсутствие обработки исключений во время финализации. Если финализатор генерирует исключение, процесс финализации останавливается, оставляя объект в поврежденном состоянии без какого-либо уведомления.
- Финализаторы также влияют на переносимость программы. Поскольку алгоритм сборки мусора зависит от реализации JVM, программа может отлично работать в одной системе, но вести себя по-разному в другой.
- Снижается производительность, так как JVM приходится выполнять гораздо больше операций при создании и уничтожении объектов.

<aside>
💡

начиная с Java 9, функция `finalize` устарела и в конечном итоге будет удалена.

</aside>

**`Атака финализаторов`**

- Базовый класс выполняет **проверки безопасности** в своём конструкторе.
- Злоумышленник создаёт **подкласс**, вызывая `super()` с некорректными параметрами.
- Конструктор базового класса выбрасывает исключение.
- Несмотря на это, метод `finalize()` подкласса может «воскресить» объект.

```java
class SecureResource {

    public SecureResource(String password) {
        if (!"secret".equals(password)) {
            throw new SecurityException("Access denied");
        }
    }
}

class Attack extends SecureResource {
    public static Attack instance;

    Attack () {
        super("wrong"); // Исключение в конструкторе
    }

    @Override
    protected void finalize() {
        instance = this; // Воскрешение объекта
    }
}
```

Чтобы защититься от этой атаки можно сделать следующее:

- cделать класс final
- запретить переопределение finalize() , объявив метод final с пустой реализацией
- использовать статические фабричные методы
